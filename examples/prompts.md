# Prompt Examples

下面这些 prompt 可以直接复制到 Codex 或支持 Skill/Agent 文件读取的模型环境中使用。

## 业务观察

```text
Use $interflowai-koubo-script

我的想法：我发现很多公司做 AI 落地，差距不是技术，而是一号位有没有亲自把业务问题拆清楚。
目标用户：创始人、品牌操盘手、AI 落地负责人
这个想法来自哪里：一天内见了两个不同行业的一号位，他们都在亲自判断 AI 应该进到业务哪里。
我想让用户相信什么：AI 落地不是先找工具，而是先识别哪段业务值得被重做。
我手上的证据或素材：会议笔记、客户业务链路草图、AI 工具界面截图、当天复盘文档。
希望用户评论什么：你们公司现在是老板亲自抓 AI，还是技术部门在做项目？
这条内容最好留下什么资产：一号位 AI 诊断的开场判断。
```

## 流程干货

```text
Use $interflowai-koubo-script

我的想法：我把口播生产从靠灵感改成了一套录制前工作流，先判断选题，再生成前 5 秒、口播稿、素材清单和复盘表。
目标用户：自媒体创作者、品牌内容运营、小团队主理人
这个想法来自哪里：自己做内容时发现，最耗脑的不是写稿，而是判断哪条值得拍、怎么组织证据。
我想让用户相信什么：高质量口播不靠临场发挥，录制前就能把结构准备好。
我手上的证据或素材：输入卡、输出模板、生成结果截图。
希望用户评论什么：你最想自动化口播生产里的哪一步？
这条内容最好留下什么资产：口播工作流模板索取入口。
```

## 活动复盘

```text
Use $interflowai-koubo-script

我的想法：我参加一个线下 AI 活动后发现，大家不再只问模型能力，而是在问 AI 怎么进入真实业务流程。
目标用户：AI 创业者、品牌运营、企业创新负责人
这个想法来自哪里：线下活动现场、嘉宾分享、观众提问和会后交流。
我想让用户相信什么：AI 活动真正有价值的不是信息量，而是能不能带走一个可执行的业务问题。
我手上的证据或素材：现场照片、议程截图、几个高频问题、会后合作线索。
希望用户评论什么：你参加 AI 活动最想带走工具、案例，还是业务机会？
这条内容最好留下什么资产：下一场工作坊主题和潜在合作线索。
```

## 只有一句想法

```text
Use $interflowai-koubo-script

我只有一句想法：很多内容创作者不是不会写，而是不知道哪条内容值得先拍。

请先补齐必要假设，再按 skill 流程输出完整口播内容包。
```

## 本地模型复刻

如果模型环境不支持 Skill 文件夹，可以把 `SKILL.md`、`references/`、`assets/output-template.md` 拼成一个长 prompt，再粘贴测试输入。

要求模型只输出结果，不输出读取文件过程。
