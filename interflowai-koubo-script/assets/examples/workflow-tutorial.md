# 样例：流程干货

## 输入

```text
我的想法：我把口播生产从靠灵感改成了一套录制前工作流，先判断选题，再生成前 5 秒、口播稿、素材清单和复盘表。
目标用户：自媒体创作者、品牌内容运营、小团队主理人
这个想法来自哪里：自己做内容时发现，最耗脑的不是写稿，而是判断哪条值得拍、怎么组织证据。
我想让用户相信什么：高质量口播不靠临场发挥，录制前就能把结构准备好。
我手上的证据或素材：输入卡、输出模板、历史口播报告、生成结果截图。
希望用户评论什么：你最想自动化口播生产里的哪一步？
这条内容最好留下什么资产：口播工作流模板索取入口。
```

## 期望输出特征

- 主模型：流程干货型。
- 先讲结果，再讲旧流程痛点。
- 每一步说清输入、AI 做什么、人判断什么、产物是什么。
- 素材清单里要包含模板截图和结果截图。
