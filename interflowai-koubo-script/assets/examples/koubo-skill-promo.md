# 样例：口播 Skill 宣传稿

## 输入

```text
我的想法：我做了一个口播 Skill，帮自己把真实业务现场变成可录制的短视频口播。
目标用户：想做自媒体但开不了口的创作者、小品牌主理人、业务型创始人。
这个想法来自哪里：身边很多人明明有故事、有选题、有业务现场，但一到镜头前就讲不出来。
我想让用户相信什么：好的口播不是靠灵感硬憋，而是可以用一套录制前工作流准备出来。
我手上的证据或素材：Skill 输出截图、工作台生成结果、前后对比案例、发布后复盘表。
希望用户评论什么：你最卡的是选题、开头，还是怎么讲得像自己？
这条内容最好留下什么资产：口播 Skill 使用需求、前后对比案例线索、下一条教程主题。
```

## 期望输出特征

- 主模型：产品方法论型。
- 子结构：示范工作流型。
- 开头先讲痛点，不要先说“大家好我是”。
- 自我介绍放在痛点之后，一句话即可。
- 主体使用 3-4 个动作步骤：判断值不值得讲、切成用户有用的角度、补证据和素材、发布后复盘。
- 每一步先讲用户任务，再讲 Skill 怎么帮忙。
- 结尾用选择题：选题、开头、像自己。
- 必须输出小红书口播五维发布检查。

## 结构骨架

```text
痛点 hook：现在很多人想做口播，但卡住的不是没有内容，而是有故事也讲不出口。
身份一句话：我是 J，正在用 AI 把真实业务现场变成内容资产。
第一步：先判断这条值不值得讲。Skill 帮我看目标用户、点击理由和资产。
第二步：把自我表达切成对别人有用的角度。Skill 帮我反推封面和前 5 秒。
第三步：补证据，不让口播只剩观点。Skill 给截图、案例、流程和素材清单。
第四步：发布后复盘。Skill 看封面、开头、内容深度、互动和画面支撑。
价值观结尾：好的口播不是灵感，而是录制前工作流。
评论问题：你最卡的是选题、开头，还是怎么讲得像自己？
```
