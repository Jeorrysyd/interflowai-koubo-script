# 口播语言 DNA

口播稿不是文章，也不是报告。目标是让用户可以打开相机，顺着讲出来。

## 必须做到

- 第一句直接进入判断、结果或现场。
- 一句话只讲一个意思。
- 每 2-3 句出现一个具体对象：人、事、截图、现场、数据、对话。
- 抽象词后面要跟例子。
- 转折要自然，像真人思考，而不是论文结构。
- 结尾要给用户一个可以评论的问题。

## 不要这样写

```text
在当下这个快速变化的时代，越来越多的人开始关注 AI 对内容行业的影响。
```

改成：

```text
我最近发现，很多人不是不会用 AI 写内容，而是不知道哪条内容值得先拍。
```

## 开头规则

前 5 秒只做一件事：兑现标题或封面承诺。

好开头：

- `我最近越来越确定一件事：AI 的差距，很多时候不是技术差距，而是老板有没有亲自下场。`
- `很多小品牌不是没有内容，而是每天的业务现场没有被整理成口播素材。`
- `我这次去线下活动，真正让我意外的不是人多，而是大家问的问题都变了。`

坏开头：

- `哈喽大家好，今天想跟大家分享一个话题。`
- `最近 AI 很火，相信大家都听过。`
- `今天我来给大家介绍一下这个工具。`

## 中段规则

中段不要平均铺信息。按语义推进：

- 观点判断：现象 -> 证据 -> 原因 -> 影响 -> 建议。
- 流程干货：结果 -> 旧痛点 -> 新步骤 -> 人的判断 -> 复用。
- 活动复盘：现场细节 -> 洞察 -> 对谁重要 -> 下一步。
- 产品方法论：结果 -> 问题 -> 方法 -> 结构 -> 迁移。

## 句子节奏

优先使用短句和中句。长句要拆开。

推荐节奏：

```text
先给判断。
再补证据。
然后解释这个证据为什么重要。
最后告诉用户，这跟他有什么关系。
```

## 口吻规则

- 像一个有真实经验的人在复盘。
- 不要像营销号喊口号。
- 不要像课程老师讲定义。
- 不要把每句话都写得很满，保留一点真人停顿感。

## 字幕钉子

字幕钉子是能单独被截出来的短句。每条口播生成 4-6 个。

好的字幕钉子：

- `不是缺内容，是业务现场流走了`
- `AI 工具不难，难的是业务判断`
- `先判断值不值得拍，再写稿`
- `截图不是装饰，是信任证据`

坏的字幕钉子：

- `欢迎大家一起交流`
- `这是一个很重要的话题`
- `希望对你有帮助`
