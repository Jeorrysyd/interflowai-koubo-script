# 证据和资产规则

高表现口播不是把观点说得更响，而是让用户相信“这个人真的见过、做过、判断过”。

## 证据类型

优先从这些证据里选择 2-4 个：

- 现场证据：活动、会议、客户拜访、工作坊、路演、展台、白板。
- 对话证据：客户原话、创始人提问、评论区问题、团队讨论。
- 过程证据：文档、表格、流程、SOP、工具界面、生成结果。
- 结果证据：入围、成交、交付、节省时间、产出数量、反馈。
- 外部证据：新闻、行业变化、公开案例、平台趋势。
- 反例证据：踩坑、失败稿、错误流程、用户不买账的原因。

## 证据链结构

每条口播至少建立一条证据链：

```text
我看到了什么 -> 它说明什么 -> 对目标用户有什么影响 -> 现在应该怎么做
```

例：

```text
我一天内见了两个行业的一号位。
他们都没有把 AI 当成技术部门项目。
这说明真正的 AI 落地，先发生在业务机会判断里。
所以创始人不能只问“用什么工具”，要先问“哪段业务值得被重做”。
```

## 素材清单

输出口播稿时，必须告诉用户录制前准备什么：

- 真人主镜头要讲什么。
- 哪 2-3 张截图能证明观点。
- 哪个现场或对话细节要保留。
- 哪些词适合做大字幕。
- 哪些内容适合放评论区或置顶补充。

## 内容资产判断

每条口播都要留下至少一个资产：

- 一个可复用判断。
- 一个客户或用户问题。
- 一个真实案例。
- 一个后续选题。
- 一个模板、清单或方法论。
- 一个合作、诊断、工作坊或交付入口。

如果一条内容只有表达，没有资产，标记为风险。

## 隐私和公开边界

默认不输出敏感姓名、客户隐私、未公开商业数据或私人聊天素材细节。需要公开时，改成：

- `某品牌创始人`
- `一个线下活动现场`
- `一位客户的问题`
- `一张内部流程截图的脱敏版`

不要为了增强真实感暴露不该公开的信息。
